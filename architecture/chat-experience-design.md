# 心动投递 - 聊天体验设计方案

## 📋 概述

**设计理念：** 让聊天自然流动，不尴尬，有温度，像老朋友一样  
**核心目标：** 提升匹配双方的聊天积极性，降低沟通门槛，促进关系发展

---

## 1. 智能开场白推荐系统

### 1.1 个性化开场白生成

#### 基于匹配结果的开场白

```typescript
interface OpeningLineContext {
  matchScore: number           // 匹配分数
  commonInterests: string[]    // 共同兴趣
  sharedValues: string[]       // 共同价值观
  lifestyleMatch: string[]     // 生活方式匹配
  personalityMatch: string     // 性格匹配度描述
}
```

**开场白分类：**

| 类型 | 示例 | 适用场景 |
|------|------|----------|
| **共同兴趣型** | "看到你也喜欢徒步，最近有去过什么好玩的地方吗？" | 有共同爱好 |
| **价值观共鸣型** | "我们都把家庭放在第一位，这点让我很有共鸣" | 价值观高度匹配 |
| **轻松好奇型** | "你的旅行照片超赞！是在哪里拍的呀？" | 照片/资料吸引 |
| **真诚直接型** | "其实我很少主动开口，但看到你的资料觉得很想认识你" | 高匹配度 |
| **幽默型** | "系统说我们很配，要不我们来验证一下算法准不准？" | 年轻用户 |

#### 破冰问题推荐

**系统预设破冰问题池：**

**AI 动态生成逻辑：**

```typescript
function generateAIOpening(context: OpeningLineContext): string {
  const prompt = `
    你是一个温暖、真诚的朋友，正在和刚认识的潜在对象聊天。
    
    背景：
    - 匹配度：${context.matchScore}%
    - 共同兴趣：${context.commonInterests.join('、')}
    - 共同价值观：${context.sharedValues.join('、')}
    
    请生成一个自然、不尴尬的开场白，要求：
    1. 15-30字，简洁有力
    2. 带有具体细节，不要泛泛而谈
    3. 包含开放式问题，方便对方回复
    4. 语气轻松，像朋友一样
  `
}
```

### 1.2 开场白展示策略

**三个推荐选项：**
- **最佳推荐**：基于最高匹配维度生成
- **备选一**：基于第二匹配维度
- **备选二**：轻松幽默型（降低压力）

**展示时机：**
- 匹配成功后立即展示（首次进入聊天）
- 用户首次打开聊天界面时
- 5分钟内未发送消息时再次提醒

## 2. 话题引导系统

### 2.1 冷场检测机制

```typescript
interface SilenceDetection {
  lastMessageTime: Date       // 最后一条消息时间
  silenceDuration: number     // 沉默时长（分钟）
  messageCount: number        // 当前消息总数
  averageResponseTime: number // 平均回复时间
}
```

**触发条件：**
- 单方沉默超过 3 分钟 → 轻度提醒
- 双方沉默超过 5 分钟 → 话题推荐
- 连续 3 次简短回复（"嗯"、"好的"、"哈哈"）→ 话题引导

### 2.2 话题卡片系统

#### 话题来源

1. **用户资料衍生**
   - 共同兴趣扩展："你们都喜欢旅行 → 推荐旅行话题"
   - 互补兴趣探索："一个爱运动一个爱看书 → 互相分享"

2. **热门话题池**
   ```typescript
   const hotTopics = {
     seasonal: [
       "春天来了，有什么踏青计划吗？",
       "最近有什么好看的电影推荐吗？",
     ],
     trending: [
       "看到你也在北京，最近天气不错，周末有安排吗？",
     ],
     universal: [
       "你最喜欢的解压方式是什么？",
       "如果有一天的完全自由，你会怎么安排？",
     ],
   }
   ```

3. **AI 动态生成**
   - 基于聊天历史分析
   - 基于双方资料特征
   - 基于当前热点事件

#### 话题卡片组件设计

```
┌────────────────────────────────────┐
│ 💡 话题推荐                         │
│                                    │
│ 🎵 "你们都喜欢音乐，聊聊最近在听什么？" │
│                                    │
│ [发送这个话题] [换一个]             │
└────────────────────────────────────┘
```

**卡片类型：**
- **兴趣共鸣卡**：基于共同兴趣
- **问答卡**：有趣的互动问题
- **回忆卡**：童年、青春话题
- **假设卡**："如果"系列问题
- **分享卡**：推荐对方可能喜欢的

### 2.3 话题轮换策略

**避免尴尬：**
- 同一话题不重复推荐
- 话题之间有逻辑递进
- 根据聊天氛围调整话题深度

**智能排序：**
```typescript
function rankTopics(
  topics: Topic[],
  context: ChatContext
): Topic[] {
  return topics.sort((a, b) => {
    // 优先级：共同兴趣 > 用户资料相关 > 热门话题 > 通用话题
    const scoreA = calculateTopicScore(a, context)
    const scoreB = calculateTopicScore(b, context)
    return scoreB - scoreA
  })
}
```

## 3. 聊天氛围优化

### 3.1 情感分析系统

```typescript
interface SentimentAnalysis {
  tone: 'positive' | 'neutral' | 'negative'
  emotions: {
    joy: number      // 开心
    interest: number // 兴趣
    confusion: number // 困惑
    awkward: number   // 尴尬
  }
  suggestions: string[]
}
```

**应用场景：**
- 检测到尴尬氛围 → 推荐轻松话题或表情包
- 检测到积极氛围 → 鼓励继续深入
- 检测到消极情绪 → 温柔提醒或暂停话题推荐

### 3.2 回复建议系统

**场景分类：**

| 场景 | 建议类型 | 示例 |
|------|----------|------|
| 不知如何回复 | 话题延续 | "那你呢？你一般喜欢..." |
| 对方分享好消息 | 热情回应 | "哇，恭喜！怎么做到的？" |
| 对方表达困惑 | 共情支持 | "能理解，我也有过类似的感受..." |
| 聊天陷入死胡同 | 温柔转向 | "对了，我最近发现一个很有趣的事..." |

**回复建议展示：**
- 用户输入框为空且停顿 5 秒后显示
- 最多显示 3 个建议
- 建议可一键发送或修改

### 3.3 纪念日提醒

```typescript
interface Milestone {
  type: 'first_match' | 'first_message' | 'first_meeting'
  date: Date
  daysSince: number
  message: string
}
```

**纪念日类型：**
- 认识第 7 天："你们认识一周啦！🎉"
- 认识第 30 天："一个月了，你们聊了很多有趣的话题"
- 对方生日提醒
- 特殊节日提醒（情人节、圣诞节等）

**展示方式：**
- 聊天界面顶部横幅
- 轻量级，不打断对话

### 3.4 幽默元素融入

**表情包推荐：**
- 根据聊天内容推荐相关表情包
- 时机：轻松话题、化解尴尬、表达开心

**趣味道具：**
- "真心话"邀请卡
- "问问对方"小纸条
- "同频时刻"标记

## 4. 游戏化互动设计

### 4.1 真心话大冒险（轻量级）

**设计原则：**
- 问题经过精心设计，避免敏感/尴尬
- 双方自愿参与
- 可随时退出

**问题分级：**
```typescript
const truthQuestions = {
  light: [
    "你最近一次发自内心的笑是什么时候？",
    "你觉得自己的哪个优点最被低估？",
  ],
  medium: [
    "什么事情会让你特别有成就感？",
    "你理想中的完美约会是什么样的？",
  ],
  deep: [
    "你觉得爱情中最重要的是什么？",
    "有什么事是你很想做但不敢做的？",
  ],
}
```

**游戏流程：**
```
用户A发起 → 选择问题类型 → 系统随机出题 → 用户A回答 → 邀请用户B回答
```

### 4.2 问答游戏

**基于问卷延伸：**
```typescript
interface QuizGame {
  source: 'questionnaire' // 基于问卷数据
  questions: QuizQuestion[]
  format: 'guess' | 'compare' | 'discuss'
}
```

**游戏类型：**
1. **猜猜看**：猜对方的答案
   - "你觉得TA会选择稳定还是冒险？"
   - 揭晓答案，增加了解

2. **观点匹配**：看你们有多相似
   - 同时作答同一问题
   - 对比答案，讨论差异

3. **深度讨论**：聊价值观
   - 基于问卷核心问题
   - 引导深入对话

### 4.3 共同任务

**设计理念：** 创造"共同经历"感

**任务类型：**
| 任务 | 描述 | 时长 |
|------|------|------|
| 一起听歌 | 分享一首歌，同步听 | 3-5分钟 |
| 共同歌单 | 创建两人共享的歌单 | 持续 |
| 电影推荐 | 互相推荐一部电影 | 即时 |
| 读书分享 | 分享最近在读的书 | 即时 |
| 周末计划 | 讨论这周末可以做什么 | 即时 |

### 4.4 默契度测试

**触发时机：** 聊天超过 50 条消息

**测试内容：**
- 10 个选择题
- 问题基于双方资料和聊天历史
- 生成"默契报告"

**报告展示：**
```

- 都重视家庭 ❤️
- 都喜欢户外运动 🏃
```

## 5. 安全与边界保护

### 5.1 敏感话题检测

**检测范围：**
- 骚扰性语言
- 过度隐私询问
- 诈骗风险词汇
- 负面情绪表达

**处理方式：**
```typescript
interface SafetyAction {
  level: 'warn' | 'guide' | 'block'
  response: string
  followUp?: string
}
```

**分级响应：**
- 轻度：温柔引导转移话题
- 中度：提示边界提醒
- 重度：提供举报入口

### 5.2 边界提示系统

**场景化提示：**
| 场景 | 提示内容 |
|------|----------|
| 过快推进关系 | "慢慢来，好的感情需要时间培养 ✨" |
| 询问敏感信息 | "有些事可以等更熟悉后再聊哦" |
| 表达不适 | "如果感到不舒服，可以随时说出来" |

### 5.3 举报与保护

**快捷举报入口：**
- 聊天界面悬浮按钮
- 举报类型选择
- 一键拉黑

**保护机制：**
- 举报后立即停止对话
- 人工审核介入
- 保护举报者隐私

### 5.4 聊天礼仪引导

**首次聊天提示：**
```
💡 聊天小贴士：
- 保持真诚和尊重
- 有问有回，双向互动
- 感到不适可以随时说明

祝你们聊得愉快！💕
```

## 6. 聊天数据分析

### 6.1 聊天活跃度统计

```typescript
interface ChatAnalytics {
  totalMessages: number
  messagesPerDay: number
  activeHours: number[]
  engagementScore: number
}
```

**数据展示：**
- 聊天天数
- 消息总数
- 最活跃时间段
- "话痨指数"（趣味展示）

### 6.2 关系进展分析

**进展阶段：**
1. **初识期**（0-3天）：主要使用开场白和话题推荐
2. **探索期**（4-14天）：话题深入，游戏化互动
3. **熟悉期**（15-30天）：分享日常生活
4. **亲密期**（30天+）：建议线下见面

**进展建议：**
- 根据阶段智能推荐功能
- 适时提醒关系进展
- 提供见面建议

### 6.3 隐私保护

**数据存储：**
- 聊天内容加密存储
- 分析在本地/加密环境进行
- 用户可删除聊天记录

**展示原则：**
- 数据仅供用户自己查看
- 不展示具体内容分析
- 以鼓励性语言为主

## 8. 设计原则总结

### 核心原则

1. **自然不尴尬**
   - 推荐内容像朋友建议，不是机器指令
   - 给用户选择权，不是强制

2. **有温度**
   - 语言温暖、真诚
   - 尊重用户感受

3. **有趣不幼稚**
   - 游戏化设计精致、有质感
   - 幽默感恰到好处

4. **安全可靠**
   - 边界保护优先
   - 敏感检测及时

5. **渐进式引导**
   - 不急于求成
   - 让关系自然发展

### 避免的陷阱

❌ 过度干预，打扰正常聊天  
❌ 推荐内容千篇一律  
❌ 幽默感用力过猛  
❌ 安全提示过于机械  
❌ 数据分析侵犯隐私
