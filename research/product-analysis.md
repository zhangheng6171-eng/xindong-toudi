## 1. 产品概述

### 1.1 产品定位

**Date Drop** 是一款基于深度问卷匹配的校园约会应用，核心特点：

- **"心动投递"机制**：每周定期匹配，制造期待感
- **匿名社交**：保护用户隐私，降低社交压力
- **精准匹配**：66道心理问卷，科学配对
- **校园场景**：聚焦大学生群体，建立信任基础

### 1.2 目标用户画像

| 维度 | 描述 |
|------|------|
| **年龄层** | 18-26岁（大学生、研究生） |
| **需求** | 寻找认真恋爱关系、扩大社交圈 |
| **痛点** | 传统约会软件太肤浅、社交圈子有限、害怕被拒绝 |
| **价值观** | 重视精神契合、愿意投入时间了解对方 |

### 1.3 核心价值主张

> **"让每一次心动都有回响"**

- 降低初次接触的心理门槛
- 提供高质量匹配而非海量推荐
- 创造浪漫的相遇仪式感

### 2.1 66道问题设计框架

#### 2.1.1 问题分类体系

#### 2.1.2 问题设计原则

**心理学依据：**

1. **大五人格模型 (Big Five)**
   - 开放性 (Openness)
   - 尽责性 (Conscientiousness)
   - 外向性 (Extraversion)
   - 宜人性 (Agreeableness)
   - 神经质 (Neuroticism)

2. **价值观一致性理论**
   - 共同价值观是长期关系稳定的核心
   - 差异过大导致冲突，过于相似缺乏吸引力
   - 寻找"互补中求同"的平衡点

3. **依恋理论 (Attachment Theory)**
   - 安全型、焦虑型、回避型
   - 依恋类型匹配影响关系质量

**问题设计技巧：**

```markdown
✅ 好问题示例：
"周末你更喜欢如何度过？"
A. 独自在家看书/追剧
B. 和几个好友聚会
C. 尝试新的户外活动
D. 陪伴家人

❌ 避免的设计：
- 双重否定："你不喜欢不做计划吗？"
- 社会期许偏差："你认为诚实重要吗？"
- 过于抽象："你是个好人吗？"
```

### 2.2 匹配算法核心逻辑

#### 2.2.1 多维度加权匹配

```python
# 伪代码示例
def calculate_match_score(user_a, user_b):
    """
    计算两个用户的匹配分数
    """
    scores = {
        'values': calculate_values_similarity(user_a.values, user_b.values),
        'personality': calculate_personality_compatibility(user_a.personality, user_b.personality),
        'interests': calculate_interest_overlap(user_a.interests, user_b.interests),
        'lifestyle': calculate_lifestyle_match(user_a.lifestyle, user_b.lifestyle),
        'expectations': calculate_expectation_alignment(user_a.expectations, user_b.expectations)
    }
    
    # 加权计算
    weights = {
        'values': 0.30,        # 价值观最重要
        'personality': 0.25,   # 性格次之
        'interests': 0.20,     # 兴趣
        'lifestyle': 0.15,     # 生活方式
        'expectations': 0.10   # 恋爱期望
    }
    
    total_score = sum(scores[k] * weights[k] for k in scores)
    
    # 差异性调整（避免过度相似）
    diversity_bonus = calculate_diversity_bonus(user_a, user_b)
    total_score += diversity_bonus * 0.05
    
    return min(total_score, 100)
```

#### 2.2.2 匹配策略矩阵

| 维度 | 高相似度 | 中等相似度 | 低相似度 |
|------|---------|-----------|---------|
| **价值观** | ✅ 高匹配分 | ⚠️ 需其他维度弥补 | ❌ 不推荐 |
| **性格** | ✅ 容易相处 | ✅ 互补空间大 | ⚠️ 需谨慎 |
| **兴趣** | ✅ 共同话题 | ✅ 探索新领域 | ⚠️ 可能无聊 |
| **生活方式** | ✅ 减少冲突 | ⚠️ 需磨合 | ❌ 易产生摩擦 |

#### 2.2.3 冷启动策略

**新用户匹配优先级：**

1. **前3周**：优先匹配活跃度高的用户
2. **前10次匹配**：提升匹配质量权重
3. **首次匹配**：确保 ≥85分的高质量配对

## 3. 每周匹配机制

### 3.1 匹配周期设计

```
一周循环：
┌─────────────────────────────────────────────────┐
│                  周日 20:00                      │
│              ⏰ 新匹配发布通知                    │
│         "你的心动投递已送达，快来看看！"          │
└───────────────────┬─────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────┐
│              周一 ~ 周六                         │
│         💬 与匹配对象聊天互动                    │
│         📸 解锁更多信息/照片                     │
│         💝 发起邀约/告白                         │
└───────────────────┬─────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────┐
│              周日 19:00                         │
│         📊 本周互动数据汇总                      │
│         🔄 下周匹配预告                          │
└───────────────────┴─────────────────────────────┘
```

### 3.2 匹配数量策略

| 用户状态 | 每周匹配数 | 说明 |
|---------|-----------|------|
| **新用户（1-3周）** | 2-3个 | 高质量匹配，建立信心 |
| **活跃用户** | 3-5个 | 平衡数量与质量 |
| **匹配成功率低** | 增加到5个 | 补偿机制 |
| **用户设置偏好** | 1-7个 | 个性化定制 |

### 3.3 匹配推送心理学

**"期待效应"应用：**

1. **预告通知**
   - 周日18:50发送："还有10分钟，你的心动即将送达..."
   - 制造期待感，增加打开率

2. **开箱体验**
   - 匹配揭晓时的动画效果
   - 类似"开盲盒"的惊喜感

3. **稀缺性营造**
   - "本周只为你精选了3位"
   - 强调质量而非数量

## 4. 特色功能分析

### 4.1 爱神模式 (Cupid Mode)

#### 4.1.1 功能定义

**爱神模式** 是一种匿名助攻功能，允许用户：
- 帮朋友推荐合适的匹配对象
- 匿名表达对某人的好感
- 创造"意外"相遇的机会

#### 4.1.2 使用场景

```
场景1：助攻好友
┌─────────────────────────────────┐
│ 👫 A觉得B和C很般配              │
│    A开启"爱神模式"              │
│    系统检测B和C的匹配度         │
│    若匹配度>75%，推荐成功       │
│    B和C收到："有位爱神为你们牵线"│
└─────────────────────────────────┘

场景2：匿名告白
┌─────────────────────────────────┐
│ 💕 A暗恋B，但不敢表白           │
│    A使用"暗恋告白"功能          │
│    B收到："有人暗恋你"          │
│    若B也对A有意，系统牵线       │
│    双向确认后，身份揭晓         │
└─────────────────────────────────┘
```

#### 4.1.3 隐私保护机制

| 保护措施 | 说明 |
|---------|------|
| **单向匿名** | 只有双向确认后才揭晓身份 |
| **反骚扰** | 每人每周最多3次使用 |
| **举报机制** | 可举报不当使用 |
| **时效性** | 告白72小时未回应自动过期 |


#### 7.2.1 匹配算法服务

```python
# 匹配服务架构
class MatchingService:
    """
    匹配算法核心服务
    """
    
    def __init__(self):
        self.value_matcher = ValueMatcher()      # 价值观匹配
        self.personality_matcher = PersonalityMatcher()  # 性格匹配
        self.interest_matcher = InterestMatcher()  # 兴趣匹配
        self.lifestyle_matcher = LifestyleMatcher()  # 生活方式匹配
    
    async def find_matches(
        self, 
        user_id: str, 
        count: int = 3,
        exclude_ids: List[str] = None
    ) -> List[MatchResult]:
        """
        为用户寻找匹配对象
        
        Args:
            user_id: 用户ID
            count: 返回匹配数量
            exclude_ids: 排除的用户ID列表
        
        Returns:
            匹配结果列表
        """
        # 1. 获取用户档案
        user_profile = await self.get_user_profile(user_id)
        
        # 2. 获取候选池
        candidates = await self.get_candidate_pool(
            user_profile, 
            exclude_ids
        )
        
        # 3. 并行计算匹配分数
        match_tasks = [
            self.calculate_match(user_profile, candidate)
            for candidate in candidates
        ]
        match_results = await asyncio.gather(*match_tasks)
        
        # 4. 排序并返回Top N
        match_results.sort(key=lambda x: x.score, reverse=True)
        return match_results[:count]
    
    async def calculate_match(
        self, 
        user: UserProfile, 
        candidate: UserProfile
    ) -> MatchResult:
        """
        计算两个用户的匹配分数
        """
        # 各维度匹配
        value_score = await self.value_matcher.match(
            user.values, 
            candidate.values
        )
        personality_score = await self.personality_matcher.match(
            user.personality, 
            candidate.personality
        )
        interest_score = await self.interest_matcher.match(
            user.interests, 
            candidate.interests
        )
        lifestyle_score = await self.lifestyle_matcher.match(
            user.lifestyle, 
            candidate.lifestyle
        )
        
        # 加权总分
        total_score = (
            value_score * 0.30 +
            personality_score * 0.25 +
            interest_score * 0.20 +
            lifestyle_score * 0.15
        )
        
        # 计算共同点
        commonalities = self.find_commonalities(user, candidate)
        
        return MatchResult(
            user_id=candidate.id,
            score=total_score,
            value_match=value_score,
            personality_match=personality_score,
            interest_match=interest_score,
            lifestyle_match=lifestyle_score,
            commonalities=commonalities
        )
```

```python
# 每周匹配定时任务
from apscheduler import AsyncScheduler

class WeeklyMatchingJob:
    """
    每周匹配任务
    """
    
    def __init__(self, matching_service: MatchingService):
        self.matching_service = matching_service
        self.notification_service = NotificationService()
    
    async def run(self):
        """
        执行每周匹配
        """
        # 1. 获取本周需要匹配的用户
        users = await self.get_active_users()
        
        # 2. 批量匹配
        match_results = {}
        for user in users:
            matches = await self.matching_service.find_matches(user.id)
            match_results[user.id] = matches
        
        # 3. 存储匹配结果
        await self.save_match_results(match_results)
        
        # 4. 发送通知
        for user_id, matches in match_results.items():
            if matches:
                await self.notification_service.send_match_notification(
                    user_id,
                    len(matches)
                )
    
    # 定时: 每周日20:00执行
    @scheduler.scheduled_job('cron', day_of_week='sun', hour=20, minute=0)
    async def scheduled_matching():
        job = WeeklyMatchingJob(matching_service)
        await job.run()
```

```python
from fastapi import WebSocket

class ChatService:
    """
    实时聊天服务
    """
    
    def __init__(self):
        self.connections: Dict[str, WebSocket] = {}
        self.message_store = MessageStore()
    
    async def connect(self, user_id: str, websocket: WebSocket):
        """
        建立WebSocket连接
        """
        await websocket.accept()
        self.connections[user_id] = websocket
        
        # 发送离线消息
        offline_messages = await self.message_store.get_offline(user_id)
        for msg in offline_messages:
            await websocket.send_json(msg)
    
    async def disconnect(self, user_id: str):
        """
        断开连接
        """
        if user_id in self.connections:
            del self.connections[user_id]
    
    async def send_message(
        self, 
        from_user: str, 
        to_user: str, 
        content: str
    ):
        """
        发送消息
        """
        message = {
            "from": from_user,
            "to": to_user,
            "content": content,
            "timestamp": datetime.utcnow().isoformat()
        }
        
        # 存储消息
        await self.message_store.save(message)
        
        # 实时推送
        if to_user in self.connections:
            await self.connections[to_user].send_json(message)
        
        # 触发信息解锁检查
        await self.check_info_unlock(from_user, to_user)
```
